Punto A.

Se realizo el despliegue de un servidor Linux Centos en Azure. En su despliegue se realizo la creacion de una red independiente en la cual se habilitaron
los puerto de entrada 22, 80 y 8088.

******************* Instalar apache *******************************
1) sudo su
2) yum update httpd -- Actualizacion de paquetes
3) yum -y install httpd -- Instalacion Apache
4) systemctl start httpd -- subir el servicio de apache
5) systemctl status httpd -- Validar el servicio
6) Ingreso a la ruta /etc/httpd/conf y en el archivo httpd.conf le cambio el puerto a 8088 en la linea "listen"
7) systemctl start httpd -- subir nuevamente el servicio de apache
8) systemctl status httpd -- Validar el servicio nuevamnete
9) se valido con el comando firewall-cmd --state si el firewall esta deshabilitado y/o habilitado.

******************** Problemas*********************************
Al momento de cambiar el puerto de 80 a 8088 me generaba un error "make_sock: could not bind to address [::]:8088" luego de investigar
encontre que el SELinux solo permitía que apache/httpd a puertos especificos, la solucion fue ingresar a editar el archivo config en la ruta
/etc/selinux y en la lina donde indica SELINUX dejarlo con "Disable".
